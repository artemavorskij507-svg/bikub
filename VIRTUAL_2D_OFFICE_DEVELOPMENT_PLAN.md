## 🔄 Пайплайн разработки

### Принцип работы:
```
Агент → Задача → Реализация → Тестирование → Проверка → Следующий агент
```

### Пример пайплайна для задачи "Создать API для управления агентами":

1. **@backend-architect** — спроектировать API
   - Deliverable: OpenAPI спецификация
   - Передача: @security-engineer

2. **@security-engineer** — проверить безопасность
   - Deliverable: Отчёт об уязвимостях
   - Передача: @frontend-developer

3. **@frontend-developer** — реализовать UI
   - Deliverable: React компоненты
   - Передача: @api-tester

4. **@api-tester** — протестировать API
   - Deliverable: Отчёт о тестировании
   - Передача: @performance-benchmarker

5. **@performance-benchmarker** — нагрузочное тестирование
   - Deliverable: Отчёт о производительности
   - Передача: @code-reviewer

6. **@code-reviewer** — код-ревью
   - Deliverable: Отчёт о качестве кода
   - Передача: @technical-writer

7. **@technical-writer** — документация
   - Deliverable: API документация
   - Завершение задачи
