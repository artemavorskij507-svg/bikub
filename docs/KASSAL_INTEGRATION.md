# Интеграция с Kassal API

## Описание

Интеграция с Kassal API для синхронизации магазинов и продуктов из норвежской базы данных магазинов.

## Настройка

### 1. Добавьте API ключ в `.env`

```env
KASSAL_API_KEY=your_api_key_here
```

### 2. Запустите миграцию

```bash
php artisan migrate
```

Это добавит поле `kassal_id` в таблицы `retail_stores` и `products`.

## Использование

### Синхронизация всех данных

```bash
php artisan kassal:sync
```

### Синхронизация только магазинов

```bash
php artisan kassal:sync --stores-only
```

### Синхронизация только продуктов

```bash
php artisan kassal:sync --products-only
```

## Структура

- **Конфигурация**: `config/kassal.php`
- **Сервис**: `app/Services/KassalSyncService.php`
- **Команда**: `app/Console/Commands/KassalSyncCommand.php`

## Особенности

- Автоматическое создание/обновление записей по `kassal_id`
- Обработка ошибок и логирование
- Валидация данных перед сохранением
- Поддержка частичной синхронизации (только магазины или только продукты)

## Логирование

Все ошибки синхронизации записываются в `storage/logs/laravel.log`.

## API Endpoints

Сервис использует следующие endpoints Kassal API:
- `GET /api/v1/physical-stores` - список магазинов
- `GET /api/v1/products` - список продуктов

## Примечания

- Убедитесь, что API ключ действителен и имеет необходимые права доступа
- Синхронизация может занять некоторое время в зависимости от объема данных
- Рекомендуется запускать синхронизацию через cron или планировщик задач
