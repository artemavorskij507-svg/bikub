## 📊 Технические детали:

### Реализованные модели:
- **Route**: маршруты с метаданными и эффективностью
- **RouteStop**: остановки с ETA и уверенностью
- **PaymentEvent**: события платежей с идемпотентностью
- **ScheduleSlot**: обновлен с capacity/booked
- **Task**: обновлен с checkin/checkout и методами валидации

### API контракты (реализованы):
```json
POST /api/v1/routes
{
  "date": "2025-10-28",
  "vehicle_id": null,
  "orders": ["uuid1", "uuid2"]
}

PATCH /api/v1/routes/{id}/stops
{
  "stops": [
    {"order_id": "uuid1", "seq": 1},
    {"order_id": "uuid2", "seq": 2}
  ]
}

GET /api/v1/dispatch/state
// Возвращает полный снимок: orders, routes, slots, couriers
```

### Оптимизации:
- **Кэширование**: Dispatch state кэшируется на 30 сек
- **Дельта-обновления**: Готовность к WebSocket событиям
- **Батч-операции**: Группировка заказов по зонам
- **Индексы**: route_stops(route_id, seq), orders(scheduled_at, status)
